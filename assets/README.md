# `assets/` — block registry for the schematic loader

## `blocks.json` (optional but recommended)

Drop Pumpkin's `blocks.json` here as `assets/blocks.json`. At build time,
`build.rs` compiles it into an embedded `block name + properties -> global state
id` table (`mapping.rs`). Without it, only a tiny hand-written fallback set of
blocks (stone, dirt, logs, glass…) maps correctly — enough to smoke-test, not
enough for real schematics.

Where to get it: it ships in the Pumpkin repo under `assets/blocks.json`, or is
generated by Pumpkin's data extractor for your server's Minecraft version. Use
the file that matches the **same Minecraft version your server runs**, otherwise
state ids will be off.

Supported shapes (build.rs auto-detects):
- a bare array: `[ { "name": "...", "default_state_id": N, "states": [ {"id": M}, ... ] }, ... ]`
- `{ "blocks": [ ... ] }`
- a mojang-style report map: `{ "minecraft:stone": { "states": [ {"id": M, "default": true, "properties": {...}}, ... ] }, ... }`

The third shape (with per-state `properties`) gives the best results: it lets the
loader resolve exact property combinations like `oak_log[axis=x]`. The other
shapes resolve property-bearing blocks to their **default** state.

## Runtime schematics

Schematic files do **not** go here. They live in the plugin's data folder at
runtime: `plugins/islands/schematics/<name>.schem`. Paste with
`/island paste <name>`.
